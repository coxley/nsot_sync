REQUIRED_ATTRS¶

list

If you’re driver sets attributes, you can’t guarantee the remote end will have these set up. To get around this, override the REQUIRED_ATTRS property. This should be a list of NSoT attribute dicts. If you’re not familiar with the format, see the below examples:

>>> REQUIRED_ATTRS = [
        {
            'name': 'dc',
            'resource_name': 'Device',
            'description': 'Datacenter',
            'display': True,
            'required': True,
            'multi': False,
        }
    ]
Parameters:click_ctx (click.Context) –

Context from Click. Context.obj values are manually set to share throughout the context at the entrypoint functions. Any that need sharing are probably passed by the main one (noop, site_id, etc) but when subclassing you know your driver best

Also helps with doing CLI exits like click_ctx.fail(msg)

class nsot_sync.drivers.simple.SimpleDriver(click_ctx=None)[source]¶

Bases: nsot_sync.drivers.base_driver.BaseDriver

Simple driver

This driver will collect network information via netifaces on the localhost then generate NSoT resources for them and the host. The only attribute created and applied is ‘desc’.

Note

If you’re writing a driver to supplement just attributes, consider subclassing this class to get the localhost network data.

REQUIRED_ATTRS¶

list

To ensure ‘desc’ is created for each resource
